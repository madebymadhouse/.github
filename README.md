## On the org naming

You'll notice the GitHub org handles look unusual: `nqyta`, `orynadus`, `autyladus` — each with a `y` where you'd expect an `i`. Here's why.

The original organizations (`nqita`, `autiladus`, `orinadus`) were tied to a predecessor GitHub account that was suspended. The suspension stemmed from an experimental agent swarm architecture that was flagged during development — an early, now-abandoned version of what was then called Autiladus. That project is gone. Autiladus itself has been completely restarted as an unrelated client services agency.

After the suspension, the original org names were no longer available. New organizations were registered with `i` replaced by `y` as a simple workaround:

| GitHub org handle | Brand name |
|-------------------|------------|
| [nqyta](https://github.com/nqyta) | Nqita |
| [orynadus](https://github.com/orynadus) | Orinadus |
| [autyladus](https://github.com/autyladus) | Autiladus |

The Y handles are GitHub-only. All documentation, copy, and UI always use the brand names as written above.
